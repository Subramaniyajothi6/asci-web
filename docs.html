Introduction

This is the companion guide to the effects playground. Everything here is written to be learnable from zero — if you can read a little HTML, you can follow along. Each idea comes with a running demo you can watch and code you can copy.

The whole playground is a single HTML file with no build step and no dependencies except a web font. That's on purpose — you can open it, read it, and change it.

ASCII animation

ASCII art animation means drawing pictures with text characters and updating them many times a second. The trick is a simple loop: build a grid of characters, print it into a <pre> tag, wait a few milliseconds, repeat.

The famous spinning donut works in three moves:

  • Make 3D points on the surface of a doughnut (a torus).
  • Rotate & project them onto a 2D grid, keeping the nearest one per cell with a z-buffer.
  • Pick a character based on how much light hits that point — dark to bright becomes .,-~:;=!*#$@.
donut.js — live

        

        
The core of a text loop is tiny:

        
const pre = document.getElementById('screen');
setInterval(() => {
  let out = '';
  for (let y = 0; y < rows; y++) {
    for (let x = 0; x < cols; x++) out += pickChar(x, y);
    out += '\n';
  }
  pre.textContent = out;   // one full frame
}, 50);

        
Everything is text — matrix rain, plasma, fire and tunnels are the same loop with a different pickChar.

      


      

        
Bitmap & dithering

        
Old screens could only show black or white — no grays. Dithering fakes shades of gray by scattering black & white dots in a clever pattern. That crunchy 1-bit look is the "bitmap aesthetic."

        
The fastest method is ordered (Bayer) dithering: compare each pixel's brightness to a threshold from a small repeating matrix. Above the threshold → light; below → dark.

        

          
bayer.dither — live

          

        

        
// 4×4 Bayer matrix, values 0–15
const B = [[0,8,2,10],[12,4,14,6],[3,11,1,9],[15,7,13,5]];

const threshold = (B[y % 4][x % 4] + 0.5) / 16;
const lit = brightness > threshold;   // true = white pixel

        
Render to a small canvas and scale it up with image-rendering:pixelated to get chunky, crisp bitmap pixels.

      


      

        
Pixel sprites

        
A sprite is a little pixel drawing. The easy way to make one: write it as ASCII art, then turn each non-empty character into a colored square.

        

          
sprite.js — live

          

        

        
Store two frames and flip between them on a timer and the sprite "walks":

        
const art = [
  "..X..X..",
  ".XXXXXX.",
  "X.XXXX.X",
  "X.X..X.X"
];
art.forEach((row, y) =>
  [...row].forEach((ch, x) => {
    if (ch === 'X') drawPixel(x, y, '#45a557');
  })
);

        
For truly choppy 8-bit motion, don't animate smoothly — jump between whole frames. See the next section.

      


      

        
WebGL shaders

        
A fragment shader is a tiny program that runs on your GPU, once per pixel, every frame. You give it the pixel's position and it returns a color. Because the GPU does thousands of pixels in parallel, it's incredibly fast.

        

          
shader.glsl — live

          

        

        
// GLSL — runs per pixel
precision highp float;
uniform vec2 r;   // canvas size
uniform float t;   // time
void main() {
  vec2 uv = (gl_FragCoord.xy - 0.5 * r) / r.y;
  float v = sin(uv.x*6.0+t) + sin(uv.y*6.0+t);
  vec3 col = 0.5 + 0.5 * cos(vec3(0,2,4) + v + t);
  gl_FragColor = vec4(col, 1.0);
}

        
Shaders are powerful but fiddly — always give a plain-canvas fallback in case WebGL isn't available.

      


      

        
Choppy motion (the steps() trick)

        
Modern animation glides. Retro animation snaps. The secret is CSS steps(), which chops an animation into a fixed number of jumps instead of a smooth slide.

        
/* smooth — modern */
.smooth { animation: move 1s linear infinite; }

/* choppy — 8-bit */
.retro  { animation: move 1s steps(4) infinite; }

        
Four steps means the element only ever sits in 4 positions — exactly the stutter that makes pixel art feel alive. The same idea drives the loading bar that fills in blocks and the sprite bob.

      


      

        
Generative graphics

        
"Generative" just means the picture is made by rules and math, not drawn by hand. You keep a list of particles (little objects with a position and velocity), nudge them every frame, and draw them. Change the rule, change the art.

        
The playground uses five variations on this one idea:

        
    
          
  • Constellation — dots drift; draw a line between any two that are close (and to your cursor).
    • 
          
  • Flow field — each particle reads an angle from a math field and follows it, leaving trails.
    • 
          
  • Fireworks — a rocket rises, then spawns dozens of particles that fall under gravity.
    • 
          
  • Orbits — particles are pulled toward a point (your mouse) like gravity.
    • 
          
  • Starfield — points rush toward you in 3D for a hyperspace warp.
    • 
        

        

          
constellation.js — move your mouse

          

        

        
// the shared shape of every generative effect
function frame() {
  ctx.clearRect(0, 0, W, H);
  for (const p of particles) {
    p.x += p.vx;  p.y += p.vy;   // move
    drawDot(p.x, p.y);            // draw
  }
  requestAnimationFrame(frame);   // ~60fps loop
}

        
For glowing trails, don't clear the canvas — paint a semi-transparent rectangle over it each frame so old frames fade out.

      


      

        
Image → ASCII

        
To turn a picture into text, you shrink it to a tiny grid, read the brightness of each cell, and swap that brightness for a character. Dark → space; bright → a dense character like @.

        

          
Shrink the image
Draw it onto a small canvas — say 80 cells wide. Each cell now averages a chunk of the picture.

          
Read brightness
For each cell, convert its RGB to a single 0–1 value: 0.299·R + 0.587·G + 0.114·B.

          
Map to a character
Use brightness to index a "ramp" string ordered dark → bright.

        

        

          
converter.js — live

          

        

        
const ramp = ' .:-=+*#%@';                // dark → bright
const b = (0.299*r + 0.587*g + 0.114*bl) / 255;
const ch = ramp[Math.floor(b * (ramp.length - 1))];

        
Want more detail? Braille characters pack a 2×4 dot grid into one glyph — 8× the resolution of one ASCII char.

      


      

        
Conway's Game of Life

        
The Game of Life is a cellular automaton: a grid of cells that are alive or dead, evolving by four tiny rules applied to every cell at once. From this, startlingly complex patterns emerge.

        
    
          
  • A live cell with 2 or 3 live neighbors survives.
    • 
          
  • A dead cell with exactly 3 live neighbors is born.
    • 
          
  • Everything else dies (loneliness or overcrowding).
    • 
        

        

          
life.js — click to draw cells

          

        

        
const alive = grid[i];
next[i] = alive
  ? (n === 2 || n === 3 ? 1 : 0)   // survive?
  : (n === 3 ? 1 : 0);              // born?

      


      

        
Fractals (Mandelbrot)

        
The Mandelbrot set is a picture of a simple formula repeated. For every pixel, you run z = z² + c over and over. If the number stays small, the pixel is "in the set" (black); if it flies off to infinity, you color it by how fast it escaped.

        

          
mandelbrot.js — live

          

        

        
let x = 0, y = 0, it = 0;
while (x*x + y*y <= 4 && it < max) {
  const xt = x*x - y*y + x0;   // z = z² + c
  y = 2*x*y + y0;
  x = xt;  it++;
}
// `it` (escape speed) → color

        
To zoom, just shrink the range of x0/y0 you scan around the click point, and raise the iteration limit for more detail.

      


      

        
Game loops

        
Every game — Snake, Breakout, anything — is the same three-beat loop running forever: read input → update the world → draw it.

        

          
State
Keep the whole game in plain variables: positions, velocities, score, "is it over?".

          
Input
Listen for keys or the mouse and change the state (e.g. turn the snake).

          
Update & draw
Move things, check collisions, then repaint. Repeat on a timer.

        

        

          
loop.js — a ball + paddle

          

        

        
function tick() {
  ball.x += ball.vx;  ball.y += ball.vy;         // update
  if (ball.x < 0 || ball.x > W) ball.vx *= -1;  // bounce
  draw();                                       // render
}
setInterval(tick, 16);   // ~60 times a second

      


      

        
Geist Pixel renderer

        
Vercel's Geist Pixel font renders letters as grids of shapes (square, circle, grid, triangle, line). You can imitate it: draw normal text to a hidden canvas, read which pixels are "on," then stamp a shape wherever a pixel is lit.

        

          
geist-pixel.js — type & pick a shape

          

            
            

              
              
              
            

            
          

        

        
// read the hidden text, stamp a shape per lit pixel
const px = ctx.getImageData(0,0,cols,rows).data;
if (px[(y*cols + x) * 4 + 3] > 128) {   // alpha = is it ink?
  ctx.beginPath();
  ctx.arc(cx, cy, r, 0, 6.28);        // draw a circle "pixel"
  ctx.fill();
}

      


      
      

        
What is Vercel?

        
Vercel is a full compute platform for building and running web apps — not just a place to host static sites. You connect a Git repo, and every push is built and deployed automatically.

        
    
          
  • It runs full backends — Next.js, Express, FastAPI, Hono and more — with zero config.
    • 
          
  • It gives every branch a preview URL so you can test before going live.
    • 
          
  • It handles scaling, caching and a global network for you.
    • 
        

        
Mental model: you write code, push to Git, Vercel turns it into a fast website with an address.

      


      

        
Deployments & preview URLs

        

          
Connect your repo
Link a GitHub/GitLab/Bitbucket project once.

          
Push a branch
Vercel builds it and gives you a unique preview URL to share and test.

          
Merge to main
That promotes the build to production. Bad release? Instant rollback to a previous deploy.

        

        
# deploy from your terminal
npm i -g vercel

vercel --prod     # production deploy

      


      

        
Functions & Fluid Compute

        
A Vercel Function is backend code (an API route) that runs on demand. Modern Vercel runs these on Fluid Compute by default — full Node.js, with instances reused across requests to kill cold starts.

        
    
          
  • Node.js 24 LTS is the current default (also Python, Bun, Rust).
    • 
          
  • Default timeout is now 300s on all plans.
    • 
          
  • Billing is Active CPU — you pay for CPU time actually used.
    • 
        

        
Old advice said "use Edge Functions." Today, prefer Fluid Compute — same regions and price, but real Node.js. Middleware supports full Node.js too.

      


      

        
Frameworks & rendering

        
Vercel builds Next.js and supports many other frameworks with zero config (SvelteKit, Nuxt, Astro, Remix…). The important idea is how a page is rendered:

        
    
          
  • Static — built once, served instantly from the edge.
    • 
          
  • SSR — rendered per request in a Function (for personalized/live data).
    • 
          
  • ISR (Incremental Static Regeneration) — static pages that quietly rebuild on a schedule or on demand. Works on Next.js, SvelteKit, Nuxt and Astro.
    • 
          
  • PPR (Partial Prerendering) — one page mixes a static shell with streamed dynamic holes.
    • 
        

        
Image Optimization resizes and reformats images automatically, and Routing Middleware lets you rewrite/redirect/personalize a request before it's cached — framework-agnostic, full Node.js.

        
Rule of thumb: static by default, ISR when data changes slowly, SSR when it must be fresh per request.

AI Gateway

        
AI Gateway is one endpoint to reach hundreds of AI models. Instead of wiring up each provider, you route through the gateway and get observability, budgets and automatic fallbacks.

        
import { generateText } from 'ai';

const { text } = await generateText({
  model: 'anthropic/claude-sonnet-5',  // provider/model string
  prompt: 'Explain dithering to a 10 year old.'
});

        
With the AI SDK on Vercel, use plain "provider/model" strings through the gateway — no per-provider packages needed.

  • Geist Sans — the proportional UI font.
    • 
          
  • Geist Mono — monospace, for code.
    • 
          
  • Geist Pixel — a pixel font with five shape variants: Square, Circle, Grid, Triangle, Line.

Color tokens

        
Geist's palette is built from named accent colors plus a neutral gray scale. Click a swatch to copy its hex.

        

        
Use tokens (variables), not raw hex, so light/dark themes and future tweaks stay in one place:

        
:root {
  --blue:   #0070f3;
  --fg:     #ededed;
  --bg:     #000000;
}
button { background: var(--blue); color: var(--fg); }
